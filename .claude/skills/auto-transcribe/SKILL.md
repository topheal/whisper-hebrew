# סריקה ותמלול אוטומטי לפי דרישה

מריץ סריקה חד-פעמית של כל התיקיות שמוגדרות ב-`config.yaml`, לפי סדר העדיפות שם. בתוך כל תיקייה, הקבצים החדשים ביותר (לפי תאריך שינוי) מתומללים ראשונים.

## הוראות

1. הרץ:
```
python "<תיקיית_הפרויקט>/auto_transcribe.py"
```

אם ההרצה נכשלת עם `ModuleNotFoundError`, התקן תלויות ונסה שוב:
```
pip install -r "<תיקיית_הפרויקט>/requirements.txt"
```

2. הצג את הפלט בזמן אמת — לכל קובץ שמטופל יודפס: התחלת תמלול, נתיב ה-SRT שנשמר, ואז סיכום (אם מופעל לתיקייה הזו).

3. בסיום — סכם למשתמש כמה קבצים טופלו ואיפה הפלט נשמר (`<שם>.srt`, ובתיקיות עם summarize: true גם `<שם>_summary.txt`, באותה תיקייה של קובץ המקור).

## הערות
- ברירת המחדל לכל תיקייה: רק תמלול SRT (מנוע מקומי, Whisper) — בלי סיכום
- סיכום אוטומטי מופעל לתיקייה ספציפית ע"י `summarize: true` באותה רשומת תיקייה ב-config.yaml
- `maxdays: N` בתיקייה ספציפית מגביל אותה לקבצים שהשתנו ב-N הימים האחרונים בלבד
- הסיכום מופק ע"י Claude Code headless (`claude -p`) - דרך המנוי הקיים, בלי מפתח API נפרד
- קבצים עם שם זהה אך סיומת שונה (אודיו+וידאו) מתומללים פעם אחת בלבד
- קבצים שכבר תומללו (יש להם .srt עדכני, ו-summary.txt עדכני אם summarize מופעל) מדולגים אוטומטית
- קבצים שעדיין מסתנכרנים (Google Drive) מדולגים ויטופלו בסריקה הבאה
- לשינוי תיקיות/תדירות/מקביליות/סיכום — ערוך את config.yaml

# תמלול עברית מקומי עם Whisper

תמלול קבצי אודיו ווידאו לעברית — **מקומי, חינמי, ללא אינטרנט בשימוש**.

מבוסס על [faster-whisper](https://github.com/SYSTRAN/faster-whisper) — גרסה מהירה פי 4 מ-Whisper המקורי.

---

## דרישות

- Python 3.8+
- ffmpeg

### התקנת ffmpeg (Windows)
1. הורד מ-[gyan.dev/ffmpeg/builds](https://www.gyan.dev/ffmpeg/builds/) — בחר `ffmpeg-release-essentials.zip`
2. חלץ לתיקייה כגון `C:\ffmpeg`
3. הוסף `C:\ffmpeg\bin` ל-PATH של Windows

---

## התקנה

```bash
git clone https://github.com/topheal/whisper-hebrew.git
cd whisper-hebrew
pip install -r requirements.txt
```

או הרץ את `setup.bat` (Windows בלבד).

## שימוש

### ממשק גרפי (מומלץ)
```bash
python transcribe_gui.py
```
לחץ על הכפתור הירוק, בחר קובץ — וזהו.

### שורת פקודה
```bash
python transcribe.py "נתיב/לקובץ.mp3"
```

הפלט נשמר אוטומטית כקובץ `.txt` באותה תיקייה של הקובץ המקורי.

## מודלים

| מודל | גודל | דיוק בעברית | מהירות |
|------|------|------------|--------|
| `small` | 500MB | סביר | מהיר מאוד |
| `medium` | 1.5GB | **טוב (ברירת מחדל)** | מהיר |
| `large-v3` | 3GB | מצוין | איטי יותר |

לשינוי מודל — ערוך את `MODEL_SIZE` בתחילת `transcribe.py`.

## שימוש עם Claude Code

העתק את תיקיות `.claude/skills/transcribe` ו-`.claude/skills/auto-transcribe` לתוך `~/.claude/skills/` שלך.

על מחשב אחד (לא לצורך הפצה ל-repo אחר) - מומלץ Junction במקום העתקה, כדי שלא ייווצרו שני קבצים שיכולים להתבדר זה מזה:
```cmd
mklink /J "%USERPROFILE%\.claude\skills\auto-transcribe" "<נתיב לפרויקט>\.claude\skills\auto-transcribe"
```
(לא דורש הרשאות מנהל, בניגוד ל-symlink רגיל)

אחר כך פשוט כתוב לקלוד: **"תמלל לי את הקובץ..."** או **"סרוק את התיקיות"**

## תמלול אוטומטי לתיקיות (Google Drive ועוד)

מערכת שסורקת תיקיות מוגדרות ומתמללת קבצים חדשים אוטומטית. לכל תיקייה אפשר לבחור בנפרד:
- `<שם>.srt` — תמלול מלא עם תזמונים מדויקים (**ברירת מחדל** - מנוע מקומי, Whisper, בלי Claude)
- `<שם>_summary.md` — בנוסף, סיכום בנקודות + תוכן עניינים מתוזמן (כל שורה: זמן + נושא, מבוסס על תזמוני Whisper עצמם), שמופק ע"י Claude Code (headless, דרך המנוי הקיים - בלי מפתח API נפרד ובלי עלות נוספת) - רק אם הופעל לתיקייה הזו

### הגדרה

`config.yaml` הוא קובץ אישי (לא נכלל ב-git, כדי שלא יחשוף את נתיבי התיקיות הפרטיות שלך אם תשתף/תדחוף את הריפו). בהפעלה ראשונה הוא נוצר אוטומטית מ-`config-default.yaml` (תבנית עם דוגמאות מוערות-החוצה בלבד). ערוך את `config.yaml`:
```yaml
folders:
  # תיקייה רגילה - רק SRT (ברירת מחדל, אין צורך לכתוב summarize בכלל), עד 30 יום אחורה
  - path: 'G:\האחסון שלי\שיחות מוקלטות'
    recursive: true
    maxdays: 30

  # תיקייה עם סיכום אוטומטי בנוסף ל-SRT, בלי הגבלת גיל קבצים
  - path: 'C:\Users\you\Downloads\שיחות חשובות'
    recursive: false
    summarize: true

  # תיקייה עם זיהוי דוברים (לשיחות זום עם כמה משתתפים)
  - path: 'C:\Users\you\Downloads\שיחות זום'
    recursive: false
    diarize: true

scan_interval_minutes: 60   # תדירות סריקה ברקע
max_parallel: 1              # כמה קבצים לתמלל במקביל
model_size: medium
```
שימוש בגרשיים יחידים `'...'` מאפשר להעתיק נתיב ישירות ממנהל הקבצים של Windows בלי לשנות אותו (גרשיים כפולים `"..."` דורשים הכפלת כל `\`).

סדר התיקיות ברשימה = סדר עדיפות (הראשונה מטופלת ראשונה). בתוך כל תיקייה, הקבצים החדשים ביותר מתומללים ראשונים.

`summarize: true` מפעיל סיכום אוטומטי לתיקייה הזו, דרך `claude -p` (Claude Code headless) - דורש שה-CLI מותקן ומחובר (`claude --version` עובד תקין). בלי הדגל - רק SRT, מהיר יותר וללא תלות ב-Claude.

קובץ הסיכום (`<שם>_summary.md`) מכיל שני חלקים, בקריאה אחת ל-`claude -p`:
```
## סיכום
- ...נקודות עיקריות...

## תוכן עניינים
0:01 גיוס קשב, התמדה, אנרגייה וזמן ליישום התכנית
1:18 מדיטציה קצרה
8:04 שיעור קצר - איך מגייסים קשב
28:17 *שאלות ותשובות*
28:20 מה עושים כשאין לי כסף
30:36 איך מעוררים מוטיבציה לשינוי הרגלים
```
שורות תוכן העניינים מובטחות להיות בסדר כרונולוגי (מהקוד, לא רק מהפרומפט) - גם אם קלוד יחזיר אותן לא מסודר.

אם אתה עורך/מגהה את `_summary.md` ידנית - השינוי שלך לא יידרס בהרצה הבאה, כל עוד לא שינית את קובץ המקור (הסקריפט בודק תאריך שינוי, ולא יוצר סיכום מחדש אם הקובץ קיים ועדכני מהמקור).

`maxdays: N` מגביל את התיקייה הזו לקבצים שהשתנו ב-N הימים האחרונים בלבד (לפי תאריך שינוי הקובץ). בלי הדגל - כל הקבצים בתיקייה נסרקים, בלי הגבלת גיל.

`recursive` - ברירת המחדל היא `true` (סורק גם תתי-תיקיות). אפשר לכתוב `recursive: false` כדי להגביל לקבצים ישירות בתיקייה בלבד.

אם הקובץ שנמצא הוא וידאו בלבד (אין קובץ אודיו תואם עם אותו שם - mp3/wav/m4a וכו') - נוצר אוטומטית גם קובץ `<שם>.mp3` באמצעות ffmpeg, לנוחות האזנה בלי לפתוח את הוידאו.

### זיהוי דוברים (diarize)

`diarize: true` בתיקייה ספציפית מתייג כל שורה ב-`.srt` (וב-`_summary.md`, אם `summarize: true` גם מופעל) עם מי דיבר: `דובר א': שלום, מה שלומך?`. שימושי לשיחות זום עם כמה משתתפים. כבוי כברירת מחדל - לא נוגע בתיקיות בלי הדגל.

מבוסס על מודל הזיהוי של ivrit.ai (`ivrit-ai/pyannote-speaker-diarization-3.1`), אבל הוא תלוי ברכיב פנימי אחד שכן דורש אישור וטוקן מ-Hugging Face (חד-פעמי, חינמי):

1. הרשמה/התחברות ב-[huggingface.co](https://huggingface.co)
2. אישור תנאי שימוש ב-https://huggingface.co/pyannote/speaker-diarization-community-1 (כפתור "Agree and access repository")
3. יצירת Access Token (סוג Read מספיק) ב-https://huggingface.co/settings/tokens
4. הגדרת הטוקן כמשתנה סביבה במחשב (PowerShell):
   ```powershell
   [Environment]::SetEnvironmentVariable("HF_TOKEN", "hf_...הטוקן שלך...", "User")
   ```
   נדרש לפתוח טרמינל/שורת פקודה חדשה אחרי זה כדי שהמשתנה יזוהה.
5. `pip install pyannote.audio` (לא נדרש אם לא משתמשים ב-`diarize`)

### הרצה

**חד-פעמית, על פי דרישה:**
```bash
python auto_transcribe.py
```
או הרץ `scan_now.bat`.

**ברקע, אוטומטית כל שעה (Windows Task Scheduler):**
```bash
register_task.bat
```
להסרה: `schtasks /delete /tn "WhisperHebrewAutoTranscribe" /f`

### התנהגות
- אם סריקה קודמת עדיין רצה (למשל לקחה יותר מ-scan_interval_minutes) - הסריקה החדשה מדלגת על עצמה ולא רצה במקביל (קובץ נעילה scan.lock; נעילה "תקועה" יותר מ-6 שעות מבוטלת אוטומטית)
- קבצים שבשמם מופיעה המילה "raw" (לא משנה אותיות גדולות/קטנות) מדולגים תמיד — נחשבים גרסת גלם, לא הגרסה הסופית
- קובץ אודיו וקובץ וידאו עם אותו שם — מתומלל פעם אחת בלבד (מועדף אודיו, מהיר יותר)
- קבצים שכבר תומללו (יש להם `.srt` עדכני, ו-`_summary.md` עדכני אם `summarize: true`) מדולגים
- קבצים שבתהליך סנכרון (Google Drive) מדולגים ויטופלו בסריקה הבאה
- אם זיהוי דוברים נכשל (למשל HF_TOKEN חסר) - התמלול ממשיך כרגיל בלי תיוג דוברים, לא נכשל הקובץ כולו
